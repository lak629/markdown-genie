# Markdown Genie - Read Everything

## Welcome

Markdown Genie is a simple, business-first way to make markdown useful for non-technical teams while keeping the output fully ingestible by technical systems and AI.

If your organization wants to capture processes, policies, reports, meeting notes, and operational data in a format that can be automated, indexed, and reused by systems such as LLMs, databases, and workflow engines, Markdown Genie is the bridge.

**GFM Compliance**: The generated markdown adheres to the GitHub Flavored Markdown (GFM) specification.

---

## Why Markdown Genie?

-   **Fast adoption for business users**: A clean, WYSIWYG experience removes the markdown learning barrier.
-   **Structured output for technology**: Content remains native markdown, so it can be parsed by existing tools, automation pipelines, and large language models.
-   **Better business data ingestion**: Teams can author narratives, decisions, and SOPs in a format that technical teams and AI systems can consume directly.
-   **Modern user experience**: Ribbon-style editing, live previews, image handling, Mermaid visuals, and tables make markdown feel familiar.

---

## What this means for your organization

Markdown Genie is designed to help business teams and technology teams work together with confidence.

### For non-technical contributors

-   No more confusing markdown syntax.
-   Use buttons for formatting, tables, links, and diagrams.
-   Keep content readable and professional.
-   Protect YAML frontmatter and preserve metadata automatically.

### For technical and automation teams

-   Source content remains clean markdown for parsing.
-   Frontmatter metadata stays separate and machine-readable.
-   Mermaid diagrams and tables can be transformed into structured knowledge.
-   Content can feed knowledge bases, automation workflows, and LLM ingestion pipelines.

---

## Core business benefits

-   **Faster documentation adoption**: Users can author content in a polished editor without markdown training.
-   **More consistent output**: Templates and guidance are easier to enforce when markdown is hidden behind a visual editor.
-   **Automated data capture**: Business content becomes easier to ingest into analytics, compliance, and automation systems.
-   **Future-ready format**: Markdown is both human-readable and machine-friendly, making it a great format for AI-first workflows.

## Product roadmap

Markdown Genie starts with a strong focus on developer-friendly tooling and business adoption.

1.  **VS Code deployment**
    
    -   Launch the first product as a VS Code custom editor.
    -   This enables writers, analysts, and process owners to author markdown in a modern visual editor.
    -   The first release is built for teams that already use VS Code or want to experiment with markdown in a polished environment.
2.  **Microsoft Word deployment**
    
    -   Next, bring the same markdown-friendly experience into Word.
    -   This will dramatically lower the adoption barrier for business users who already live in Word documents.
    -   It also opens a powerful path for enterprise teams to author markdown-ready content without leaving Word.
3.  **Additional product deployments**
    
    -   From there, we will expand into other delivery channels: browser editors, standalone apps, and embedded content creation tools.
    -   The long-term goal is one markdown experience that business, technical, and AI systems all trust.

## How to use this repo

This repository is the home of Markdown Genie product development. We are currently starting with a VSCode extention of **Markdown Genie**.

The repo is broken up into the following directories:

-   images - any images used for the product or testing. Currently just my dork icons for the product.
-   test - any files used to test the product. Mostly markdown files with different formatting.
-   vscode - this is the inital repo for the first product deployment of **Markdown Genie**. You can find detailed documentation in the README.md file in that directory.

If you are visiting this repo for the first time, start with the `README.md` for developer setup in the vscode directory, then use this README.md for the product vision and adoption strategy.
